This document includes the neccesary modifications to a default raspian image
These will have to be done manually to run the balloon mission software without errors. Maybe this will go into a script sometime later..

The following software is needed from packet sources:
	resample
	imagemagick
	i2c-tools
	libi2c-dev
	gpsd
	gpsd-clients
	libsndfile-dev
	libjpeg-dev


The following configuration has to be done:
	in /etc/modules add:
	snd_soc_bcm2708
	snd_soc_bcm2708_i2s
	bcm2708_dmaengine
	snd_soc_pcm5102a
	snd_soc_hifiberry_dac
	i2c-bcm2708
	i2c-dev

	remove blacklist i2c from /etc/modprobe.d/raspi-blacklist.conf

	enable the camera in raspiconfig

get the software:
	git clone https://github.com/loetlab-jena/xplorer.git
	make

in /etc/rc.local add gpsfake and start.sh as follows:
	# sudo -u pi gpsfake -c0.1 /home/pi/log3.nmea &
	stty -F /dev/ttyAMA0 4800
	gpsd -b -n /dev/ttyAMA0 &
	/home/pi/xplorer/start.sh &
	gpspipe -r -o /home/pi/missions/$(date +%Y%m%d-%H%M).txt &

	in /etc/inittab comment the serial console on /dev/ttyAMA0
